## 2. Meta tag strategy

- **Titles**: primary keyword first, pipe separator, brand suffix. Hard
  cap ‚â§ 60 chars so Google doesn't truncate the brand.
- **Descriptions**: 120‚Äì158 chars, natural keyword usage, include a
  concrete value prop ("refreshed every 8 hours", "110 creators ranked",
  etc). Never keyword-stuffed.
- **OG/Twitter**: auto-generated by `renderMetaBlock` in
  [scripts/prerender.ts](scripts/prerender.ts) from the same `PageMeta`
  object as the regular `<title>` + `<meta description>`. One source of
  truth per page.
- **Canonical**: always absolute, always `https://`, never trailing slash
  except `/`. Env-overridable via `SITE_URL`.
- **OG image**: per-release items use `item.image.url` (the verified
  release hero); home / `/influencers` / `/log` fall back to
  `${SITE_URL}/og-image.png` (ships in `public/`).

## 3. Schema / JSON-LD strategy

Each page ships exactly ONE JSON-LD block matching its content type. The
dev-mode fallback block in [index.html](index.html) is stripped during
prerender and replaced with the page-specific block before `</head>`.

| Page           | JSON-LD                                                             |
| -------------- | ------------------------------------------------------------------- |
| `/`            | `@graph` of `WebSite` (`@id:/#website`) + `Organization` (`@id:/#org`, with logo `ImageObject`) |
| `/influencers` | `CollectionPage` with `isPartOf` ‚Üí `/#website`, `publisher` ‚Üí `/#org` |
| `/log`         | `CollectionPage` (same refs as influencers)                         |
| `/releases/<id>` | `Article` (headline, description, datePublished, dateModified, author, publisher ref, image ImageObject, mainEntityOfPage, articleSection, keywords) **+** `BreadcrumbList` (Home ‚Üí release title) |

All release Article schemas carry `dateModified` from `feed.generatedAt`
so crawlers get a signal every time the 8h cron refreshes the feed.

## 4. H1 hierarchy

Google expects exactly one `<h1>` per page describing the page topic.
Until 2026-04-13 the brand `AI/TLDR` was the `<h1>` on every page, which
meant `/influencers` and `/log` were signaling "AI/TLDR" as their topic
instead of their actual content.

Fixed in App.tsx ‚Äî the brand renders as `<h1>` **only on the feed home**,
where "AI/TLDR" IS the page topic. On `/influencers` the brand is a `<p>`
and `<h1 class="inf-title">Top AI Influencers to Follow</h1>` is the
visible page title. Same pattern on `/log` with
`<h1 class="log-title">AI Release Changelog</h1>`.

CSS uses `text-transform: uppercase` so the visual stays brutalist while
the source text is sentence-case (what Google actually indexes).

## 6. Robots

[dist/robots.txt](dist/robots.txt) allows all crawlers on everything and
points at `${SITE_URL}/sitemap.xml`. No `Disallow` entries.

`<meta name="robots">` in the HTML shell is
`index, follow, max-image-preview:large, max-snippet:-1` ‚Äî lets Google
show the release hero image in rich results and use full-length snippets
for the explainer copy.

## 9. Maintenance playbook

**Whenever a new PAGE TYPE ships** (new route, new content surface):

1. Add a new `PageMeta` constant in `scripts/prerender.ts`.
2. Add a matching JSON-LD helper (or reuse `renderJsonLdCollectionPage`).
3. Wire it into `main()` with `injectMeta(template, META, jsonLd)`.
4. Add the page to the `buildSitemap()` call.
5. Add the route to `parseRoute()` in `src/App.tsx` and update the
   `document.title` effect.
6. Add a semantic `<h1>` to the component, and make sure the brand is
   NOT an `<h1>` when that page is active.
7. Update sections 1‚Äì3 of this file.
8. `bun run build` and verify the new `dist/<path>/index.html` has the
   right `<title>`, meta description, canonical, JSON-LD type.

**Whenever the AGENT runs a sweep** (every 8h via cron):

- `releases.json` gets rewritten. No SEO action needed ‚Äî prerender
  regenerates all release pages + sitemap on the next build.
- `sweeps.json` gets one new entry. `/log` regenerates automatically.

**Whenever the PUBLIC DOMAIN changes**:

- Update `SITE_URL` default in [scripts/prerender.ts](scripts/prerender.ts).
- Update absolute URLs hardcoded in [index.html](index.html) JSON-LD fallback.
- Update references in [CLAUDE.md](CLAUDE.md) and this file.
- Set up a 301 redirect from the old host (Cloudflare rule).
- Submit the new sitemap to Search Console, retire the old property.
- Grep for stale URLs: `grep -rn "old-host" . --include="*.{ts,tsx,json,md,html,yml}"`.

## 10. Verification commands

Quick sanity checks after any SEO change:

```bash
# Full build (tsc + vite + prerender)
bun run build

# Titles + descriptions on all index pages
grep -E "<title>|description" dist/index.html dist/influencers/index.html dist/log/index.html

# JSON-LD presence + type on each page type
grep '"@type"' dist/index.html
grep '"@type"' dist/influencers/index.html
grep '"@type"' dist/log/index.html
grep '"@type"' dist/releases/google-gemma-4/index.html

# One JSON-LD script per static page; two per release page (Article + Breadcrumb)
for f in dist/index.html dist/influencers/index.html dist/log/index.html dist/releases/google-gemma-4/index.html; do
  printf "%s: " "$f"; grep -c "application/ld+json" "$f"
done

# Sitemap sanity
grep -c "<url>" dist/sitemap.xml
head -20 dist/sitemap.xml

# No stale domain references
grep -rn "workers.dev\|blackpc-me" dist/ 2>/dev/null | head
```
