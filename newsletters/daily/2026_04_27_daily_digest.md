<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">OpenAI Privacy Filter — Open-Weight On-Device PII Redaction</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">A 1.5B-parameter open-weight model that scrubs personal data from text locally — no data leaves your machine.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
OpenAI Privacy Filter is a small, open-weight token-classification model (Apache 2.0) that detects and masks eight PII categories: names, addresses, emails, phone numbers, URLs, dates, account numbers, and API keys — running entirely on-device.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Labels the entire input in a single forward pass using a bidirectional transformer with sparse MoE (50M active out of 1.5B parameters). A 128K-token context window handles long documents without chunking; constrained Viterbi decoding produces coherent PII spans.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
96% F1 out of the box, Apache 2.0 for commercial use, runs on a laptop. Most PII scrubbing today is regex (brittle) or cloud API calls (sends the sensitive data away to redact it) — this solves both problems.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Data engineers, compliance teams, and devs building LLM pipelines that handle user data in regulated environments.
</p>
