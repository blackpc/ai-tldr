<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Anthropic's Project Glasswing — ~50 Partners, 10,000+ Vulnerabilities Found, 90.6% True-Positive Rate</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Anthropic's one-month report on its ~50-partner program to find software flaws before AI models can exploit them.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Project Glasswing is Anthropic's coordinated effort with about 50 partners to use its Claude Mythos Preview model to find and fix security flaws in critical software ahead of attackers. This is the program's first public results, covering its opening month.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Partners run Claude Mythos against their own codebases and a sweep of open-source projects. An automated scan of 1,000+ projects flagged 6,202 high- or critical-severity issues; independent assessment confirmed 1,587 as valid at a 90.6% true-positive rate.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
It is a concrete data point on whether frontier models can do useful defensive security at scale — 530 bugs have been reported to maintainers and 75 patched so far. Claude Security also enters public beta alongside a public vulnerability dashboard.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Security researchers and open-source maintainers who can now try the workflow through the new <a href="https://claude.com/product/claude-security" style="color: #f7ff00;">Claude Security public beta</a>.
</p>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Google Ships Managed Agents in the Gemini API — One Call Spins Up an Ephemeral Linux Sandbox</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">A single Gemini API call now provisions an ephemeral Linux sandbox with an Antigravity agent inside it.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Managed Agents is a new Gemini API capability that lets developers spin up production-grade AI agents without owning sandbox infrastructure — each call returns an agent running in an isolated, ephemeral Linux environment where it can reason, call tools, execute code, and browse the web.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>

Sessions can be resumed across follow-up calls with preserved state.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Microsoft Open-Sources RAMPART and Clarity — Pytest-Native Agent Red-Teaming + Pre-Code Design Review</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Two open-source tools that bake agent red-teaming and design review into the development workflow.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
<a href="https://github.com/microsoft/RAMPART" style="color: #f7ff00;">RAMPART</a> is a pytest-native framework for writing repeatable safety and security tests against AI agents. <a href="https://github.com/microsoft/clarity-agent" style="color: #f7ff00;">Clarity</a> is a planning assistant that interrogates a system design before any code is written, both MIT-licensed.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
RAMPART is built on PyRIT — developers write standard pytest cases for threat scenarios and a thin adapter gates CI/CD pass/fail. Clarity runs as a desktop app or web UI with multiple AI "thinkers" probing the design from security, human-factors, and operational angles.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Agent safety testing has been largely ad hoc. Putting red-team checks into pytest and CI lets teams catch prompt-injection and tool-misuse regressions automatically on every change, while Clarity pushes failure analysis upstream where fixes are cheaper.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
AI agent developers and security teams that want to add repeatable, CI-integrated safety tests without building the framework themselves.
</p>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">KanBots — Open-Source Desktop App Runs Claude Code and Codex Agents in Parallel Across a Kanban Board</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">A local kanban board where every card is a Claude Code or Codex agent working in its own git worktree.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
KanBots is an open-source (MIT) desktop app for macOS, Linux, and Windows that turns a kanban board into a dispatcher for AI coding agents. Each card on the board becomes a task handled by a <a href="https://claude.ai/code" style="color: #f7ff00;">Claude Code</a> or Codex CLI agent.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Each agent runs in its own git worktree so parallel tasks don't collide on the same files. The board updates live as work progresses, an autopilot mode splits work automatically, and cost analytics track spend per agent.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Coordinating several coding agents is fiddly without tooling. KanBots gives a single view to dispatch, watch, and review them while git worktrees keep each agent's changes isolated until you're ready to merge.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Developers running multiple coding agents who want a visual control surface with per-agent cost tracking.
</p>
