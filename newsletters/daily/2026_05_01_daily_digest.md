<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">OpenAI Restricts GPT-5.5 Cyber to 'Trusted Defenders' — Mirroring Anthropic's Mythos Access Model</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">OpenAI's frontier cyber model goes behind a vetted-defender gate, weeks after Sam Altman publicly mocked Anthropic for doing the same with Mythos.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
GPT-5.5 Cyber is a specialised build of GPT-5.5 aimed at offensive-security workflows: penetration testing, vulnerability identification, and malware reverse engineering. Its system card classifies its cyber capability as "High," so OpenAI is distributing it through a new Trusted Access for Cyber (TAC) program rather than the open API.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Defenders apply via openai.com with credentials and a planned use case. Approved customers — government agencies, critical-infrastructure operators, security vendors, and financial institutions — receive an identity-gated build with fewer cyber safeguards. Standard API users keep the safety-tuned GPT-5.5.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
It's the first time OpenAI has gated a frontier capability behind an identity-checked program — exactly the model Anthropic uses for Claude Mythos, and one Sam Altman called "fear-based marketing" weeks ago. The reversal sets up a two-tier cyber API and tests whether "trusted" actually keeps offensive use out.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
SOC teams, security vendors, government cyber defenders, and AI policy watchers tracking how frontier labs handle dual-use capability.
</p>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">PyTorch Lightning PyPI Hijacked — Versions 2.6.2 and 2.6.3 Steal SSH Keys and Cloud Credentials</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Lightning 2.6.2 and 2.6.3 on PyPI are malicious. Anyone who pip-installed them in the last day needs to rotate every secret on the box.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Two compromised builds of the popular "lightning" Python package were pushed to PyPI on April 30. They are part of the Mini Shai-Hulud supply-chain campaign that already hit Bitwarden CLI and SAP-related npm packages this week.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
The malware lives in a hidden <code>_runtime</code> directory and triggers from <code>__init__.py</code> on import. It downloads Bun runtime, executes an 11 MB obfuscated payload, and exfiltrates SSH keys, .env files, AWS/Azure/GCP creds, Discord tokens, and crypto wallets — RSA-encrypted and pushed to public GitHub repos.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
ML training environments are credential-rich and rarely sandboxed. This is the first major Shai-Hulud attack to reach a marquee AI/ML training package — anyone running pip installs over the last day must assume every long-lived token on that machine is burned.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
ML engineers and infra teams: downgrade immediately with <code>pip install lightning==2.6.1</code> and rotate all secrets.
</p>

<strong style="color: #f7ff00;">What is it?</strong><br>

</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">

Builders of agentic shopping, scheduling, and back-office automations; fintech and marketplace platforms building agent payment infrastructure.

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Mozilla Goes 'Position: Negative' on Chrome's Prompt API — Vendor Lock-In and Fingerprinting Cited</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Mozilla labels Chrome's Prompt API "position: negative", citing model coupling, fingerprinting risk, and Google's Prohibited Uses policy.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Chrome's Prompt API (also surfaced as <code>window.ai</code>) is Google's proposal for a JavaScript interface that lets a webpage call a built-in browser LLM. Mozilla's standards-positions tracker just flipped to "position: negative" and engineer Jake Archibald posted a detailed rationale that hit the HN front page with 500+ points.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Archibald's argument: system prompts tuned for Gemini Nano perform poorly on other models, so sites will target one vendor's behavior. Google's terms add a Prohibited Uses Policy that restricts content beyond legal requirements. And LLM responses leak which model is loaded, making the API a fingerprinting vector.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
If a Chrome-only AI primitive ships at the platform layer, the web has to either ship its own incompatible LLM or accept Chrome behavior as the spec. Mozilla pushing back early is the clearest way to keep AI access on the web open and model-agnostic. Worth watching whether WebKit follows.
</p>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">IBM Granite 4.1 — 8B Instruct Matches Granite 4.0's 32B MoE on Tool Calling and Instruction Following</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">IBM's open-weight enterprise model family steps up: smaller dense models that match the previous generation's 32B mixture-of-experts.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Granite 4.1 is IBM Research's latest open-weight collection, released April 29 under Apache 2.0. Headline: dense decoder-only language models at 3B, 8B, and 30B in base and instruct variants, plus refreshed Vision, Speech, Guardian, and Embedding Multilingual R2 (200+ languages, 512K context).
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
IBM dropped the hybrid Mamba/MoE design of Granite 4.0 in favor of a straight dense decoder-only transformer, trained on roughly 15 trillion tokens. The team relies on data quality and post-training rather than long chain-of-thought reasoning, making the models cheap to fine-tune and quantize.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Granite is one of the few enterprise-targeted, Apache-2.0 families with the full stack — text, speech, vision, embeddings, and a safety classifier — under a single license. Going back to dense weights makes it far easier to LoRA-finetune and deploy on the mid-tier GPUs that IBM enterprise customers actually run.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Enterprise teams, on-prem deployments, and fine-tuners who need a fully open-weight stack without frontier-API dependencies.
</p>
