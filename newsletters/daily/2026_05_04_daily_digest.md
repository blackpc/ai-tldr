<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">VS Code Reverts Default Copilot Co-Author Attribution After 1.2K-Point HN Backlash</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">After a 1.2K-point Hacker News thread, Microsoft reverts the VS Code default that tagged commits with a Copilot co-author trailer.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
VS Code 1.118 (April 29) shipped with a Git extension default that automatically appended a <code style="font-family: Menlo, Consolas, monospace; font-size: 13px;">Co-Authored-By: Copilot</code> trailer to any commit where Copilot suggested code — firing even when users had disabled AI features. PR #313931 reverts that default to off for the upcoming 1.119 build.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
The fix changes the <code style="font-family: Menlo, Consolas, monospace; font-size: 13px;">git.addAICoAuthor</code> default from <code style="font-family: Menlo, Consolas, monospace; font-size: 13px;">'all'</code> to <code style="font-family: Menlo, Consolas, monospace; font-size: 13px;">'off'</code> and gates attribution on the existing <code style="font-family: Menlo, Consolas, monospace; font-size: 13px;">chat.disableAIFeatures</code> flag, so users who already opted out of AI features no longer get co-author lines added.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
With the U.S. Copyright Office holding that non-human authors cannot hold copyright, legal teams flagged the opt-out default as a copyright-protection risk. The reversal restores opt-in attribution before 1.119 ships.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
VS Code users on 1.118, repo owners worried about silent AI attribution in their commit history.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">NIST CAISI Evaluation: DeepSeek V4 Pro Lags U.S. Frontier by ~8 Months Across Five Domains</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">NIST's CAISI publishes its first independent technical evaluation of DeepSeek V4 Pro across five capability domains.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
CAISI is the Center for AI Standards and Innovation, the AI-evaluation arm of NIST. It tested V4 Pro on nine benchmarks spanning cybersecurity, software engineering, natural sciences, abstract reasoning, and mathematics — the first US-government technical evaluation of the model.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Two of the nine benchmarks are held out to detect gaming: ARC-AGI-2's semi-private split and CAISI's internally-built PortBench. The aggregate finding: V4 Pro performs similarly to GPT-5, which shipped about eight months earlier, despite DeepSeek's own claims of near-parity with current US frontier models.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
