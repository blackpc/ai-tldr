<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Grok Automations — scheduled and email-triggered jobs land in Grok apps</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Grok can now run a saved prompt on a timer or fire it off the moment a matching email lands in your inbox.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Grok Automations save any prompt as a job that runs on its own — on a repeating schedule you pick, or when an incoming email matches a filter you set. xAI launched the feature on July 16, 2026 across grok.com and the mobile apps.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Describe the work once, choose a schedule (once, daily, weekdays, weekly, monthly, or yearly) or an email filter by sender/recipient/subject, then pick how Grok notifies you when the run finishes. Every run creates a fresh conversation and drops it into run history.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Grok moves from a chat tool you open on demand into a background worker — a recurring research digest can arrive before you wake up, a customer email summarized the second it hits your inbox. Scheduled runs are free on every tier.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Grok users who want recurring or event-driven jobs done without babysitting a chat. Email triggers require SuperGrok.
</p>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Roblox Build — mobile AI turns a text prompt into a playable game</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">A text prompt on your phone now compiles into a playable Roblox game — public alpha starts July 28 in New Zealand.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Build is a new AI creation mode inside the Roblox mobile app. Type a plain-English description of the game you want and Build produces a first playable version you can then edit, test, and share.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Prompts route through Roblox's Cube 3D foundation model, Procedural Models for parametric assets, and third-party models for gameplay logic, art style, and sound. Build shares its back end with desktop Roblox Studio so a mobile creator can jump to full Studio later without losing state.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
With ~380 million monthly active users, Roblox drops the barrier from "learn Studio on a laptop" to "type a sentence on your phone." AI-made games still compete on retention-based ranking, so low-effort slop won't surface on the homepage.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Roblox creators without a PC, and studios watching how a mainstream mobile-first text-to-game tool ships. Verified users aged 9+ to create; 16+ to publish globally.
</p>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Anthropic makes Fable 5 permanent in Max and Team Premium</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Simon Willison walks through Anthropic's July 18 reversal — Fable 5 stays in Max and Team Premium plans instead of moving to credits.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Starting July 20, Claude Fable 5 is bundled into all Max and Team Premium subscriptions at 50% of each plan's weekly usage limits. Pro and Team Standard users lose bundled access but receive a one-time $100 credit.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Under Anthropic's original plan, Fable 5 was leaving paid subscriptions on July 19 and continuing only via prepaid usage credits at $10 input / $50 output per million tokens. Competitive pressure from GPT-5.6 Sol and Kimi K3 reversed the decision.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
For Max and Team Premium subscribers, Fable 5 stops being a usage-credit purchase and goes back to being part of the plan they already pay for. Willison flags the trade-off: keeping Fable 5 in front of paying users at 50% of limits chews up GPU capacity meant for training.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Claude Max and Team Premium subscribers, and anyone tracking how frontier-lab pricing responds to open-weight competition.

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Simon Willison — Kimi K3 and the pelican benchmark come apart</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Simon Willison retires the pelican benchmark as a comparative metric — GLM-5.2 draws a better one than models that beat it everywhere else.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Willison's July 16 post runs his "pelican riding a bicycle" SVG prompt against Kimi K3, Moonshot's new 2.8T MoE model. The SVG rendered cleanly but cost ~25 cents and consumed 13,241 reasoning tokens.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
The pelican bundled three signals: capability check (can it produce valid SVG?), cost estimation (reasoning-token burn), and comparative ranking. The ranking breaks because GLM-5.2's pelican looks better than Kimi K3's, Fable 5's, and GPT-5.6 Sol's — despite losing on real benchmarks.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
The pelican was probably the most-cited informal LLM benchmark on X. Willison keeps it as a reproducible "hello world" sanity check and cost signal, but retires it as a model leaderboard.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
ML practitioners and voices who track frontier-model evaluation and informal benchmarking.
</p>
