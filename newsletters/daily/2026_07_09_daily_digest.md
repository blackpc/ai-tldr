<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Microsoft Flint — visualization language that lets AI agents author charts</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Compact chart language that AI agents can generate, humans can edit, and Vega-Lite, ECharts, or Chart.js can render.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
<a href="https://github.com/microsoft/flint-chart" style="color: #f5f5f0; text-decoration: none;">Microsoft Flint</a> is an intermediate chart specification language from Microsoft Research: agents describe a chart in a few lines, and the Flint compiler fills in scales, axes, and layout from the data and 70+ semantic types.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Every field gets a semantic type (Rank, Temperature, Price, Country, etc.) and the compiler uses that plus data cardinality to choose formatting, then emits a native <a href="https://vega.github.io/vega-lite/" style="color: #f7ff00; text-decoration: none;">Vega-Lite</a>, ECharts, or Chart.js spec. An MCP server (<code style="font-family: Menlo, Consolas, monospace; background: #181818; padding: 2px 4px;">flint-chart-mcp</code>) lets agents preview charts interactively.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Chart generation is a common failure mode for coding agents — verbose backend specs are hard to produce reliably. A shorter, semantic-first spec lets smaller models produce good-looking charts on the first try, and the same spec stays human-editable.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Developers building agentic data-analysis or dashboard tools; Microsoft Research users of Data Formulator.
</p>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">OpenAI retracts SWE-Bench Pro — audit finds ~30% of coding tasks broken</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">OpenAI audited the coding benchmark it recently told the community to use, found ~30% of tasks broken, and pulled its recommendation.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
OpenAI Research published <a href="https://openai.com/index/separating-signal-from-noise-coding-evaluations/" style="color: #f5f5f0; text-decoration: none;">"Separating signal from noise in coding evaluations"</a>, reporting that about 30% of <a href="https://scale.com/leaderboard/swe_bench_pro_public" style="color: #f7ff00; text-decoration: none;">SWE-Bench Pro</a>'s 731 public tasks have design or grading bugs — and retracting its earlier recommendation to use it as a primary coding eval.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
The audit runs two tracks: an automated flagger reviews prompts and hidden tests, then Codex-based investigator agents dig into flagged tasks inside the real repo environment. Five experienced human engineers also annotate each flagged task independently — they overlapped with the agent pipeline on 74% of categories.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Coding benchmarks drive how labs, buyers, and press rank frontier models. If SWE-Bench Pro is ~30% broken, published gains like 23.3% → 80.3% in eight months conflate real model progress with benchmark bugs — a serious credibility problem for the whole leaderboard ecosystem.
</p>
