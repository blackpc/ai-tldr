<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Inkling — Thinking Machines' first open-weights 975B/41B multimodal MoE</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Mira Murati's Thinking Machines Lab ships its first foundation model, and puts the full weights on HuggingFace under Apache 2.0.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Inkling is a 975-billion-parameter Mixture-of-Experts model that activates 41 billion parameters per token. Text, images, and audio go in; text comes out — running on a 1-million-token context window under Apache 2.0.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Each forward pass routes tokens through 6 of 256 experts across 66 decoder layers, trained on 45 trillion multimodal tokens on NVIDIA GB300 systems with the Muon optimizer plus over 30 million RL rollouts to shape reasoning.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Thinking Machines says outright that Inkling is not the strongest model available — the pitch is customizability. Enterprises get a legally fine-tunable open base; Thinking Machines gets a business around adapting it via their Tinker platform.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
ML researchers, enterprise ML teams, and open-weights users who want a multimodal MoE base they can fine-tune and self-host.
</p>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">AIDE² — first evidence of recursive self-improvement in AI R&D</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">An AI autoresearch agent rewrote its own code across 100 steps and beat a hand-tuned baseline that took two years to build.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
AIDE² pairs an outer "meta" agent that edits code with an inner autoresearch agent that runs experiments. Weco AI ran the loop for 100 iterations across 8 days, producing 7 successively better agent versions on ML engineering, GPU-kernel writing, and harness engineering.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
The outer loop (Claude Opus 4.7) modifies the inner agent's source code; the inner loop (Gemini 3-Flash) runs on benchmarks under a fixed compute budget. A change is kept only when the new agent scores higher — drift is filtered out mechanically.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
AIDE85 beats an internal baseline hand-tuned by engineers for two years on three external benchmarks the loop never trained against, so the gains transfer rather than overfit. Weco calls this Level 1 RSI — net-positive but not yet self-igniting.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
ML researchers, autoresearch and agent builders, and alignment researchers tracking recursive self-improvement signals.
</p>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Talk to Spotify — conversational AI beta lands for Premium users</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Spotify Premium users can now hold a real chat with the app to pick songs, learn about albums, and revisit their listening history.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Talk to Spotify is a text-and-voice chat on the Home and Now Playing screens. Users can request songs, ask about their listening history, get album backstories, or discover similar artists and podcasts — no menu-hunting required.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Requests go through a mix of Spotify's own AI models and outside providers, selected per task. The assistant reads Spotify's catalog, editorial context, and each user's private listening history, so answers can cite specific tracks and first-play dates.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Spotify has ~293 million Premium subscribers, so a conversational front end reaches a scale most standalone AI apps can't match. It turns music discovery into open-ended dialogue instead of typed queries or a curated stream.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Spotify Premium users age 18+ in the US, Ireland, and Sweden on iOS and Android. English only at launch.
</p>
