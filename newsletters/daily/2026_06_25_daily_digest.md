<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Gemini 3.5 Flash gets Computer Use — native browser, mobile, and desktop agents</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Gemini 3.5 Flash can now see a screen and click, type, and scroll on its own through a single built-in tool.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Computer Use is now a first-class tool on <a href="https://blog.google/innovation-and-ai/models-and-research/gemini-models/introducing-computer-use-gemini-3-5-flash/" style="color: #f7ff00;">Gemini 3.5 Flash</a>, so the same Flash model that handles chat and coding can now drive a browser, an Android phone, or a desktop without a separate model.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
The tool takes screenshots from the target environment, returns a structured action (click, type, scroll, swipe) plus an intent field explaining each step, and loops until the task is complete via the <a href="https://ai.google.dev/gemini-api/docs/computer-use" style="color: #f7ff00;">Gemini API</a> or Gemini Enterprise Agent Platform.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Enterprise teams no longer have to stitch a planning model and a separate computer-use model together — one API call now handles long-horizon work like software testing, form filling, and back-office automation. Google says this is its best performance yet on agentic computer-use tasks.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Developers and enterprises building browser, mobile, or desktop agents.
</p>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">OpenAI Jalapeño — first custom inference chip, built with Broadcom</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">OpenAI's first piece of custom silicon — an inference ASIC co-designed with Broadcom for cheaper, lower-power model serving.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
<a href="https://openai.com/index/openai-broadcom-jalapeno-inference-chip/" style="color: #f7ff00;">OpenAI Jalapeño</a> is a custom inference ASIC built for serving large language models rather than training them — co-designed by OpenAI and Broadcom, manufactured by Celestica, with engineering samples already running GPT-5.3-Codex-Spark at target frequency and power.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
The chip cuts data movement and rebalances compute, memory, and networking around the LLM inference loop, paired with Broadcom Tomahawk silicon for large-scale fabric. OpenAI used its own models to accelerate the nine-month design cycle.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Broadcom CEO Hock Tan cites roughly 50% cost savings versus typical AI GPUs on inference. OpenAI frames Jalapeño as the first step in an "Apple of AI" strategy — owning the full stack from chip to product — targeting deployment by end of 2026.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
OpenAI infrastructure teams and any developer relying on OpenAI inference economics.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Qualcomm buys Modular for $3.9B, picking up a hardware-agnostic AI compiler stack aimed at Nvidia's CUDA moat.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Qualcomm will acquire <a href="https://www.modular.com/blog/qualcomm-to-acquire-modular" style="color: #f7ff00;">Modular</a> in an all-stock deal worth ~$3.9B, taking ownership of the <a href="https://www.modular.com/mojo" style="color: #f7ff00;">Mojo language</a> and the <a href="https://www.modular.com/max" style="color: #f7ff00;">MAX AI platform</a> — built by LLVM/Swift creator Chris Lattner to compile AI models across CPUs, GPUs, and NPUs.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Up to 19.2 million Qualcomm shares go to Modular shareholders; the deal closes in H2 2026 pending regulatory approval. MAX compiles AI workloads to Nvidia, AMD, Intel, or Qualcomm hardware without a rewrite — the same model targets any chip.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Modular's mission was to break CUDA lock-in; putting MAX under a major chipmaker gives a silicon vendor full ownership of a portable AI software layer — though it also raises questions about whether a Qualcomm-owned Modular can remain truly hardware-neutral.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
AI infrastructure engineers, Mojo and MAX users, Qualcomm hardware customers, and Nvidia-alternative watchers.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>

</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Paste an API key into Copilot app settings; it stores the secret in the OS credential store and never displays it in the UI. From then on Copilot routes prompts to that provider's endpoint, letting you mix frontier and local models across tasks.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Copilot users blocked by compliance or cost from sending code to GitHub-hosted models now have an official supported path — including running a fully local model on the same laptop. Anthropic Claude becomes a first-class Copilot backend without any proxy.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Enterprises with key/provider restrictions, devs running local LLMs, and Anthropic API customers.
</p>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">GitHub Copilot CLI GA — tabbed terminal with MCP, skills, and plugins</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Copilot CLI's new tabbed terminal hits GA with Session, Issues, PRs, and Gists tabs plus in-session MCP, skills, and plugin commands.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
<a href="https://github.blog/changelog/2026-06-23-copilot-cli-new-terminal-interface-is-generally-available/" style="color: #f7ff00;">GitHub Copilot CLI's redesigned interactive terminal</a> — previewed at Microsoft Build 2026 — is now generally available, opening inside any Git repository with four tabs: Session, Issues, Pull requests, and Gists.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Slash commands inside the live session handle everything: <code style="font-family: Menlo, Consolas, monospace; background: #181818; padding: 2px 4px;">/mcp add</code> wires up MCP servers, <code style="font-family: Menlo, Consolas, monospace; background: #181818; padding: 2px 4px;">/skills</code> and <code style="font-family: Menlo, Consolas, monospace; background: #181818; padding: 2px 4px;">/plugin</code> install tool packs, and the UI switches on screen-reader support automatically. Update with <code style="font-family: Menlo, Consolas, monospace; background: #181818; padding: 2px 4px;">copilot update</code>.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Copilot CLI users no longer drop out of the terminal to triage issues or edit a config file to add an MCP server — what used to be two or three context switches now stays in one place. Theme-aware colors and screen-reader support arrive without manual setup.

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Krea AI open-sources a 12B Diffusion Transformer image model with a Turbo variant that draws 2K images in two seconds.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
<a href="https://www.krea.ai/blog/krea-2-technical-report" style="color: #f7ff00;">Krea 2</a> is a 12B Diffusion Transformer trained from scratch for aesthetic-first text-to-image generation, published as two open-weight checkpoints: Krea 2 Raw (for LoRA fine-tuning) and <a href="https://huggingface.co/krea/Krea-2-Turbo" style="color: #f7ff00;">Krea 2 Turbo</a>, an 8-step distilled engine.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
The 12B DiT backbone pairs with a Qwen3-VL text encoder using multi-layer feature aggregation. Krea 2 Turbo is distilled to 8 steps, reaching ~2 seconds per 2K image. Day-one support is live in ComfyUI, FAL, AWS, and GCP.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
An independent lab shipping a 12B open-weight image model at 2 seconds per 2K image tightens the gap with proprietary APIs. Because Raw is published, studios can fine-tune Krea 2 for their own style without retraining from scratch.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Image generation researchers, studios fine-tuning a base model, and ComfyUI workflow builders.
