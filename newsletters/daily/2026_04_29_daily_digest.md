<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">OpenAI's frontier models reach a second hyperscaler less than 24 hours after the Microsoft exclusivity ended.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Ineffable Intelligence — David Silver Leaves DeepMind, Raises $1.1B for a 'Superlearner'</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">The researcher behind AlphaGo and AlphaZero just emerged from stealth with the largest European seed in history.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
A London-based research lab founded by David Silver — who led the RL team at Google DeepMind for over a decade — co-founded with DeepMind alumni Czarnecki, Espeholt, and Junhyuk Oh. The $1.1B seed at a $5.1B post-money valuation is co-led by Sequoia and Lightspeed with Nvidia, Google, Index, BOND, EQT, and DST Global participating.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Silver wants to skip pre-training and human-generated data altogether — models would learn purely from experience inside simulations, as AlphaZero learned chess and Go from self-play. He calls the target a 'superlearner': pure reinforcement learning at frontier scale.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
It's the most prominent DeepMind departure since Anthropic's 2021 spinout, and the largest European seed in history. The bet that experience-based RL beats pre-training-and-fine-tune could pull serious talent and capex away from the dominant LLM playbook.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
RL researchers, AI investors, and anyone tracking the post-LLM frontier.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Warp's full client is now AGPL on GitHub, with OpenAI sponsoring the agent-driven contribution workflow.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
Warp's full Rust client codebase moves to a public GitHub repo under AGPL-3.0, with UI framework crates carved out as MIT so other tools can reuse them. The backend services remain closed; the project has 31,428 stars.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Contributions flow through Oz, Warp's cloud agent orchestrator powered by OpenAI's GPT family: agents propose, plan, test, and ship changes under maintainer review rather than a traditional PR queue.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
It's the first venture-backed agentic dev environment to open-source its full client while treating 'humans managing fleets of agents' as the canonical contribution model. AGPL means downstream commercial wrappers must publish their changes; MIT on UI crates invites adoption without that constraint.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Dirac — Open-Source Coding Agent Tops Terminal-Bench-2 on Gemini 3 Flash Preview</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">A solo-built Cline fork takes #1 on Terminal-Bench-2 with a Flash-tier model — the harness, not the model, was the bottleneck.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
An open-source CLI and VS Code coding agent forked from Cline by Dirac Delta Labs. Running on Gemini 3 Flash Preview, it scored 65.2% on the public Terminal-Bench-2 leaderboard — ahead of closed-source Junie CLI (64.3%) and Google's own harness baseline (47.8%) on the same model.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Three key tricks stack: hash-anchored edits identify code regions by content hash so the model never re-reads unchanged context; AST-native patching emits syntactically valid edits without retries; multi-file batching collapses several reads or edits into a single tool round-trip.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
It's a clean data point that the agent harness — not the underlying model — is gating Terminal-Bench scores: a Flash-tier model in a tight harness beat agents running on much more expensive backends.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Developers who want a low-cost CLI coding agent and harness authors looking for ideas worth copying.
</p>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">AISLE's AI Analyzer Finds 38 CVEs in OpenEMR — Two CVSS 10.0, Used by 100K Healthcare Providers</h2>

<h2 style="margin: 0 0 8px; font-size: 24px; font-weight: 800; color: #f5f5f0; line-height: 1.2;">Auto-Architecture: An Autonomous Loop Optimizes a RISC-V CPU 92% in 9 Hours</h2>

<p style="margin: 0 0 20px; font-size: 15px; color: #8a8a85; line-height: 1.5;">Karpathy's autoresearch idea, but the optimization target is a CPU's RTL instead of a neural net.</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">What is it?</strong><br>
An open experiment by FeSens that points a coding-agent research loop at a textbook 5-stage in-order RV32IM SystemVerilog core. In 9h 51m, the agent tested 73 microarchitectural hypotheses, accepted 10, and raised CoreMark/MHz from 301 to 577 iterations/s — a 92% improvement over the baseline.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">How does it work?</strong><br>
Each round, an LLM (Claude or Codex) drafts a hypothesis and implementation in an isolated worktree. A verifier stack runs riscv-formal proofs, Verilator simulation, and FPGA synthesis to check correctness and performance — hypotheses that pass formal verification AND raise CoreMark/MHz get merged; the rest are rejected.
</p>

<p style="margin: 0 0 16px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Why does it matter?</strong><br>
Most autoresearch demos have stayed inside Python and gradient descent. This is a concrete public example of the same loop applied to RTL — a domain where the verifier is hard but exists. The author's takeaway: the frontier isn't the loop, it's the verifier you can plug into it.
</p>

<p style="margin: 0 0 20px; font-size: 15px; color: #f5f5f0; line-height: 1.6;">
<strong style="color: #f7ff00;">Who is it for?</strong><br>
Hardware engineers experimenting with AI agents, autoresearch hobbyists, and anyone curious how Karpathy's recipe generalizes outside ML.
</p>
